# Circly 사용자 플로우 가이드

## 개요
사용자 관점에서 앱의 모든 화면과 버튼, 기능 실행 과정을 상세히 정리한 플로우 가이드입니다. Gas 앱의 직관적인 플로우를 참고하여 Circly만의 매끄러운 사용자 여정을 설계했습니다.

## 🚪 앱 진입 플로우

### A. 초대 링크를 통한 진입
```
📱 카카오톡/인스타그램에서 초대 링크 클릭
↓
🌐 브라우저에서 "앱 설치" 페이지 로딩
[설치하기 버튼] → App Store/Google Play로 이동
↓
📲 앱 설치 후 자동으로 앱 실행

```

## ✅ 질문 안전 정책

Circly의 핵심 질문은 외모/분위기/호감/크러시/인기/설렘을 포함한다. 단, 모든 질문은 선택받은 사람이 기분 좋아지는 칭찬형이어야 한다.

허용 예시:
- "가장 잘생긴 사람은?"
- "우리 Circle 대표 고양이상은?"
- "첫인상이 제일 설레는 사람은?"
- "웃을 때 분위기가 확 밝아지는 사람은?"
- "은근 인기 많을 것 같은 사람은?"
- "짝사랑 고백을 많이 받아봤을 것 같은 사람은?"

금지 예시:
- "가장 못생긴 사람은?"
- "제일 인기 없을 것 같은 사람은?"
- "연애 못할 것 같은 사람은?"
- "친구로만 지내고 싶은 사람은?"
- "고백하면 차일 것 같은 사람은?"

운영 원칙:
- 사용자가 자유 텍스트로 질문을 만들 수 없고, 운영자가 검수한 템플릿만 사용한다.
- 외모/호감 질문은 허용하되 비하, 조롱, 낙인, 부정 비교는 금지한다.
- 성별 제한이 필요한 질문은 템플릿과 후보 필터가 함께 동작해야 한다.

## 🗳️ 연속 투표 세션 플로우

Home 또는 Circle 상세에서 투표 세션을 시작하면 사용자는 진행 중인 질문을 4지선다 카드로 연속 처리한다. 후보 선정은 서버가 담당하며, 클라이언트는 Circle 멤버 전체를 내려받아 임의 샘플링하지 않는다.

### 후보 선정 정책
- 후보는 해당 Poll의 Circle 멤버로 제한한다.
- 현재 투표자와 투표 생성자는 후보에서 제외한다.
- 같은 Circle에서 받은 누적 득표 수가 적은 멤버를 우선 노출해 특정 친구에게만 표가 몰리지 않게 한다.
- 후보는 기본 4명이며, 섞기 버튼은 서버 후보 API를 다시 호출한다.

### 후보 부족 플로우
```
GET /polls/{pollId}/candidates
↓
status = NOT_ENOUGH_CANDIDATES
↓
투표 카드 대신 초대 우선 화면 표시
- "선택할 친구가 부족해요"
- "4지선다 투표를 만들려면 이 Circle에 후보가 더 필요해요."
- Primary CTA: Circle 초대하기
- Secondary CTA: 이 질문 건너뛰기
```

후보 부족은 오류가 아니라 핵심 플로우 상태로 취급한다. 사용자는 친구를 더 초대하거나 해당 질문을 건너뛰고 다음 질문으로 이동할 수 있어야 한다.

### 세션 완료 및 쿨다운 플로우
```
마지막 질문 투표 또는 건너뛰기
↓
POST /polls/sessions/{sessionId}/advance 또는 /skip
↓
서버가 users.next_session_at = 현재 시각 + 1시간 저장
↓
세션 완료/cooldown 화면 표시
- "오늘 라운드는 여기까지"
- 다음 세션까지 남은 시간
- Primary CTA: 친구 초대하기
- Secondary CTA: 알림 켜기
- Tertiary CTA: 홈으로 가기
```

쿨다운은 클라이언트 타이머가 아니라 서버 `GET /polls/sessions/availability` 응답을 기준으로 판단한다. 사용자가 쿨다운 시작 이후 속한 Circle에 신규 멤버를 초대해 실제 가입이 확인되면 서버가 쿨다운을 해제하고 즉시 다음 세션을 시작할 수 있게 한다. 알림 CTA는 다음 세션 가능 시점 또는 새 투표/받은 하트 알림을 받기 위한 권한 요청으로 제공한다.
